# SessionSearch v1.1 — Features & Tests Design Spec

A polish pass addressing feature gaps from the gap analysis and filling test coverage holes. Scope: 7 feature improvements and 5 test additions.

## Features

### 1. Keyword Highlighting

**File:** `PopoverView.swift`

The FTS5 `snippet()` function wraps matched terms in `<<` / `>>` markers. Currently the view strips these markers and renders plain text.

Replace the plain `Text` in `resultRow` with an `AttributedString`-based renderer. Parse the snippet for `<<...>>` pairs: text inside markers renders in gold (the app's orange accent color, `Color(red: 0.93, green: 0.55, blue: 0.24)`), text outside renders in `.secondary`.

Implementation: a helper function `highlightedSnippet(_ raw: String) -> Text` that splits the string on `<<`, then splits each segment on `>>`. The portion before `>>` is highlighted, the portion after is normal. Returns a composed `Text` view using `+` concatenation.

### 2. Arrow-Key Navigation + Enter

**File:** `PopoverView.swift`

Add keyboard navigation to the results list using `NSEvent.addLocalMonitorForEvents(matching: .keyDown)`.

Behavior:
- **Down arrow (keyCode 125):** move `selectedID` to the next result in the list (wraps to first if at end)
- **Up arrow (keyCode 126):** move `selectedID` to the previous result (wraps to last if at start)
- **Return/Enter (keyCode 36):** copy the selected result's command to clipboard (same as single-click), show "Copied!" feedback

Install the monitor in `.onAppear` on the search view, remove it in `.onDisappear`. Store the monitor reference in `@State` to ensure proper cleanup.

### 3. Search Field Auto-Focus

**File:** `PopoverView.swift`

Add `@FocusState private var isSearchFocused: Bool` to `PopoverView`. Apply `.focused($isSearchFocused)` to the `TextField`. Set `isSearchFocused = true` in `.onAppear` on the search view.

### 4. Quit Menu Item

**File:** `PopoverView.swift`

Add a "Quit" button in the footer, right-aligned after the stats text. Styled as subtle secondary text (`.font(.system(size: 10))`, `.foregroundStyle(.tertiary)`). On tap: `NSApplication.shared.terminate(nil)`.

### 5. Empty State

**File:** `PopoverView.swift`

When the query is empty and `indexStats?.sessionCount == 0` (or `indexStats == nil`), replace the results list area with a centered empty state:
- Text: "No sessions indexed yet"
- Subtitle: "Sessions will appear after the next index run"
- Button: "Rebuild Now" — triggers `indexAll` in a detached task, then refreshes stats

### 6. Single/Double-Click Conflict Resolution

**File:** `PopoverView.swift`

Fix the conflict where single-click fires during double-click by using a delayed single-click pattern:

- Add `@State private var clickTimer: DispatchWorkItem?` to `PopoverView`
- `.onTapGesture(count: 2)`: cancel the `clickTimer`, call `openInTerminal`
- `.onTapGesture(count: 1)`: cancel any existing timer, create a new `DispatchWorkItem` that calls `copyToClipboard`, schedule it after 250ms

SwiftUI delivers higher-count gestures first, so the double-tap handler fires and cancels the pending single-tap before it executes.

### 7. Improved `humanProjectName`

**File:** new `ProjectNameResolver.swift` (extracted from `PopoverView.swift`)

Extract project name resolution into a standalone function `resolveProjectName(_ encoded: String) -> String` so it can be unit tested.

Algorithm:
1. Prepend `/` to the encoded name, replace all `-` with `/` to reconstruct a candidate path
2. Check if the candidate path exists on disk via `FileManager.default.fileExists`
3. If it exists: return `URL(fileURLWithPath: path).lastPathComponent`
4. If it doesn't exist: split the encoded name on `-`, look for the last occurrence of `Desktop` or `Documents` as a segment, and return everything after that anchor joined by `-`. If no anchor is found and there are more than 2 segments, return the last 2 segments joined by `-`. If 2 or fewer segments, return the raw encoded name

This handles the common case (real projects on Desktop/Documents) correctly via filesystem, and degrades gracefully for worktrees, deleted projects, or non-standard paths.

`PopoverView.resultRow` calls `resolveProjectName(result.project)` instead of `humanProjectName(result.project)`. The old `humanProjectName` private method is removed.

## Tests

### T1. `resolveProjectName` Tests

**File:** new `ProjectNameResolverTests.swift`

Test cases:
- `-Users-neonwatty-Desktop-my-project` where `/Users/neonwatty/Desktop/my-project` exists on disk → `"my-project"`
- Encoded name where the reconstructed path does NOT exist → falls back to anchor logic (e.g. returns segment after `Desktop`)
- Name with no known anchors and >2 segments → returns last 2 segments joined by `-`
- Single segment name → returns raw string

### T2. End-to-End `indexAll` → `search` Content Correctness

**File:** `SessionStoreTests.swift`

Index the existing fixture file via `indexAll`. Search for a term known to be in the fixture content. Verify:
- Result count is 1
- Result has correct project name
- Result has valid `lastTimestamp` (non-zero)
- Snippet contains the search term

### T3. `indexAll` Resilience

**File:** `SessionStoreTests.swift`

Create a test directory with:
- One valid `.jsonl` file (copy the existing fixture)
- One malformed file (e.g. `bad.jsonl` containing `{invalid json`)

Call `indexAll`. Verify the valid session was indexed (search returns 1 result) and no crash occurred.

### T4. `sanitizeFTSQuery` Tests

**File:** `SessionStoreTests.swift`

Test `SessionStore.sanitizeFTSQuery`:
- `"hello world"` → `"hello world"` (unchanged)
- `"test*"` → `"test"` (star stripped)
- `"a | b"` → `"a  b"` (pipe stripped)
- `"(foo)"` → `"foo"` (parens stripped)
- `"say \"hi\""` → `"say \"\"hi\"\""` (double-quotes doubled)
- `"*"` → `""` (lone star → empty)
- `"normal-query"` → `"normal-query"` (hyphens preserved)

### T5. ISO8601 Fallback

**File:** `JSONLParserTests.swift`

Create a minimal JSONL string with timestamps lacking fractional seconds (e.g. `"2026-04-23T22:46:10Z"`). Parse it via `JSONLParser`. Verify `firstTimestamp` and `lastTimestamp` are non-nil and represent the correct date.
